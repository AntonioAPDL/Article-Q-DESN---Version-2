# PriceFM Application Section Polish Checklist

Date: 2026-06-25

## Objective

Polish the PriceFM application section so it reads as a self-contained
academic application rather than a local workflow report. The section should
be coherent with the notation, caution, and reproducibility standards used in
the simulation and GloFAS application sections.

## Academic-Writing Checklist

### Scope and Claims

- [x] State the empirical object before reporting results.
- [x] Identify the comparison as a selected-panel application study.
- [x] Avoid claims of broad superiority over PriceFM.
- [x] Distinguish fold-aligned PriceFM prediction comparisons from paper-wide
      benchmark headline metrics.
- [x] Use metric-specific conclusions: "lower AQL", "higher AQL", or
      "negative Q--DESN-minus-PriceFM AQL" instead of vague terms such as
      "better" or "weaker" when the metric is central.

### Model and Notation Consistency

- [x] Match the model names to the current manuscript-facing assets.
- [x] Use Q--DESN, AL, exAL, and \(\RHS_{\mathrm{NS}}\) notation consistently.
- [x] Do not mention DESN as part of the selected PriceFM panel unless the
      generated assets actually include a normal-DESN row.
- [x] Define \(\Delta\mathrm{AQL}\) once and use it consistently.
- [x] Avoid unexplained local workflow labels in the manuscript.

### Data and Protocol Clarity

- [x] Introduce the benchmark data before the model comparison.
- [x] State the sampling frequency, response scale, covariates, folds,
      quantile grid, and primary metric.
- [x] State that one validation-selected Q--DESN is used per target region and
      fold.
- [x] State that topology-neighbor covariates are part of the Q--DESN
      candidate set when available, not a separate paper-facing model.
- [x] State that Q--DESN remains a single-target readout even when its selected
      inputs include neighboring-region covariates.

### Tables and Figures

- [x] Captions should be interpretable without reading the surrounding
      paragraph.
- [x] Captions should state whether lower values or negative deltas favor
      Q--DESN.
- [x] Tables should use paper-facing names and labels, not implementation-stage
      names.
- [x] Figures should have readable labels and avoid internal workflow language.
- [x] Region-level summaries should note incomplete fold coverage when
      applicable.
- [x] Table captions should define the unit of aggregation when "rows" could
      mean either time stamps or region/fold cases.
- [x] Heatmap captions should state how missing region/fold cells are encoded.

### Reproducibility and Integration

- [x] The section should input only paper-facing PriceFM alias files.
- [x] All manuscript-facing tables and figures should be regenerated by a
      tracked script.
- [x] The manifest should record source paths, hashes, output hashes, and
      selected-panel coverage.
- [x] The forbidden-language scan should pass for `main.tex` and the
      manuscript-facing PriceFM assets.
- [x] The local PDF should compile before pushing to the Overleaf-facing main
      branch.
