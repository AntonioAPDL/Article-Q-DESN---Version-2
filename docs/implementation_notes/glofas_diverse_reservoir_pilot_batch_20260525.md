# GloFAS Diverse Reservoir Pilot Batch, 2026-05-25

This note records the eight screened reservoir candidates prepared after the
broad overnight ladder screen and the multiseed shortlist screen. The purpose
is model-selection evidence across genuinely different reservoir capacities
and shallow/deep-enough alternatives, not another D1 n300-only neighborhood.

No application fit was launched while preparing this batch.

## Candidate Rule

All candidates keep the promoted application-model contract fixed:

- Dec. 25, 2022 latent-path ensemble-likelihood Q-DESN application;
- two semantic DESN feature maps, one for the shared/reference quantile block
  and one for the GloFAS discrepancy block;
- identical reservoir specification for the two semantic maps, but different
  model inputs under the two-block latent-path design;
- independent RHS priors for the two readout coefficient blocks;
- AL working likelihood and article-side VB approximation;
- `rhs_tau0 = rhs_alpha_tau0 = 0.003`, `rhs_slab_s2 = 1.0`;
- `max_iter = 1000`, `n_draws = 2000`;
- `pi_w = 0.03`, `pi_in = 1.00`;
- launch seed `20260512`.

For \(D>1\), `n_tilde[d-1]` equals the previous layer width. This is the
current no-reduction convention: the previous layer state is passed forward,
and the next layer input map handles any dimension change.

## Interpretation

The candidates are ready for an explicit launch decision, not automatically
approved for launch. The main reason is that all accepted candidates are
`repair`, not `pass`: they are usable screened candidates, but their warnings
must be accepted as part of the launch decision. The D1 n1000 candidate is
closest to the effective-rank boundary and should be treated as a capacity-edge
candidate. The two D2 candidates are useful diversity checks, but they are more
risky than the robust D1 options.

Recommended launch order, if the next instruction is to launch candidates in
parallel:

1. launch ranks 1--5 as the core set;
2. include rank 6 as the wide shallow edge case if resources permit;
3. include ranks 7--8 as the D2 diversity set if the goal is architecture
   comparison rather than only finding the safest next manuscript run.

Do not promote any candidate until its completed fit, prediction-draw contract,
score tables, post-fit figures, and launch-readiness report have been audited.
