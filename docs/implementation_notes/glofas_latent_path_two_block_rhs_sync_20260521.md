# GloFAS Latent-Path Two-Block Sync

Date: 2026-05-21

This note records the article-side migration of the GloFAS latent-path
ensemble-likelihood application from a shared feature map to the intended
two-block Q-DESN design.

## Implemented Model Contract

For each fitted quantile level \(p_0\), the application now uses

```text
q_ref = x_ref' beta
q_glo = x_ref' beta + x_disc' alpha
```

where `x_ref` and `x_disc` are generated by separate deterministic DESN feature
blocks. The reference block is driven by the transformed reference streamflow
history. The discrepancy block is driven by `g_ret - y` in the retrospective
period.

At issued forecast horizons, the first efficient two-block implementation uses
a horizon-keyed discrepancy path

```text
d_future[h] = q_ens_p0[h] - y_future[h]
```

where `q_ens_p0[h]` is the empirical GloFAS ensemble quantile at the fitted
level. This preserves the streamed grouped AL-VB future-moment implementation.
A member-expanded discrepancy path, `g_ens_member - y_future`, remains a
documented extension because it requires member-specific future design rows and
Jacobians.

## Prior Contract

The beta and alpha coefficient blocks now receive independent regularized
horseshoe states. Existing `rhs_*` configuration fields define the beta/default
state. Optional `rhs_alpha_*` fields override the alpha state; active latent-path
configs now record those fields explicitly.

## Code Paths

- `application/R/fit_qdesn_latent_path.R`
  builds `X_beta`, `X_alpha`, keyed future designs, separate future
  linearizations, and design hashes that include both feature blocks.
- `application/R/latent_path_vb_al.R`
  combines independent beta/alpha RHS prior precisions into the joint Gaussian
  theta update and updates the two RHS states blockwise.
- `application/R/fit_qdesn_discrepancy.R`
  parses `rhs_alpha_*` overrides for AL-VB settings.
- Active latent-path configs use `feature_contract.version: "latent_path_v0.3"`
  and `two_block_design: true`.

## Validation Added

`application/tests/test_latent_path_design.R` now checks:

- distinct beta and alpha future feature matrices;
- keyed two-block future Jacobian signs;
- independent beta/alpha RHS initialization and update;
- posterior-draw prediction validation from separate feature blocks;
- dense-debug and streamed grouped future moments remain equivalent.

After the config pins were synchronized to the current shared validation
checkout, the full application test suite passed with:

```text
Rscript application/tests/run_tests.R
```

The discrepancy fit adapter test is skipped when the external Q-DESN engine is
not loaded in the test process; that skip is pre-existing and not specific to
the two-block migration.
