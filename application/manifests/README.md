# Application Manifests

This directory defines the data contract for the GloFAS application.

- `expected_schema.yaml` records the semantic columns and required properties
  expected after input ingestion.
- `input_manifest_TEMPLATE.csv` is the template for the local frozen-input
  manifest.
- `input_bundle_manifest_TEMPLATE.csv` is the registration template for a full
  frozen input bundle, including bundle-relative paths, file hashes, row
  counts, and optional inputs.
- `input_bundle_manifest.csv` is generated by
  `application/scripts/00_register_input_bundle.R` and records every configured
  bundle input, including optional inputs that are not present.
- `input_manifest.csv` is intentionally not created until the local GloFAS,
  gauge, and covariate handoffs are available. The active model-facing
  covariate input is `ppt_soil_covariates`; `climate_covariates` is retained
  only for diagnostics and provenance unless a future model contract says
  otherwise.

The workflow treats `input_manifest.csv` as required evidence. If it is
missing, `application/scripts/00_check_inputs.R` stops before any model fitting
or scoring can occur. The check stage reads the registered files and verifies
that required columns, hashes, row counts, column counts, and date ranges match
the manifest.
The generated manifest files are local audit artifacts and are ignored by git;
the templates and registration code are the tracked reproducibility contract.
